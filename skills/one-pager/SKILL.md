# One-Pager

Help PMs write feature one-pagers: lightweight alignment documents that capture the essentials without the full structure of a PRD. Ideal for quick buy-in and shared understanding.

## When to Use

- Aligning stakeholders before a full PRD
- Pitching a new feature or initiative
- Quick spec for a small scope
- When a full PRD is overkill
- Exploratory or early-stage ideas

## Process / Template

### 1. Gather Core Information

- What is the feature or initiative?
- Who is it for? (persona/segment)
- What problem does it solve?
- What does success look like? (metric or outcome)
- What's the proposed approach?

### 2. Use the One-Pager Structure

**Title & One-Liner**
- Feature name and 1-sentence description

**Problem**
- Customer problem (1–2 sentences)
- Optional: business problem or opportunity

**Proposed Solution**
- What we're building (3–5 bullets)
- Key flows or changes
- Out of scope (what we're not doing)

**Success Metrics**
- Primary metric and target
- How we'll measure

**Risks & Assumptions**
- Key assumptions we're making
- Risks or unknowns

**Timeline**
- High-level milestones (optional)
- Target launch or phase

**Open Questions**
- Decisions needed
- Dependencies to resolve

### 3. Keep It Tight

- Aim for 1 page (or equivalent in markdown)
- Bullets over paragraphs
- No more than 5–7 sections
- Skip implementation detail

### 4. Validate Alignment

- Does it answer: What? Why? Who? How we'll know?
- Can a stakeholder understand it in 2 minutes?
- Are open questions explicit?

## Output

A **One-Pager** document that:
- Fits on one page or screen
- Enables quick alignment
- Can be expanded into a PRD later
- Serves as a shared reference for the team
