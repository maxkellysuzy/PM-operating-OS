## Option C (educational / how it works)

**How we gave Cursor a “PM operating system” (and why you might want one too).**

Product work is context-heavy. Roadmaps, segments, strategy, exec updates — the AI needs to know *your* product and *your* priorities, not just syntax.

We open-sourced **PM Operating System** so that context lives in one place and gets loaded every time you work in Cursor.

**What it does:**
1. **Onboarding** — You answer questions (role, product, goals, tools). Config is written; setup generates rules, skills, and agents.
2. **Rules** — Always-on guidance: strategic framing, domain context, what to deprioritize.
3. **Skills** — On-demand: PRDs, working backwards, launch posts, experiments, one-pagers, exec updates.
4. **Agents** — Specialized tasks: feedback analysis (Slack), weekly planning (Slack + Drive), strategy review, retrospectives.
5. **Knowledge layer** — Strategy, segments, metrics. Edit markdown; agents and skills read it automatically.
6. **Memory** — Decision traces, feedback runs, weekly plans. Builds over time so the AI has temporal awareness.

Tool connections (Slack, Jira, Figma, etc.) are **optional** and asked last — you can skip and add them yourself later via Cursor Settings or the plugin marketplace.

If you use Cursor for product work, give it a try: **github.com/Sach1ng/PM-operating-OS**
